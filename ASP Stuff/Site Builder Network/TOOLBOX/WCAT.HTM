Interview with IIS Program Manager Jim Laurel

Jim Laurel is the Program Manager for Microsoft® Internet Information Server (IIS) 4.0. He supervised the production of the IIS Resource Kit, a book describing the installation and optimum use of IIS 4.0 and two accompanying CDs of code, utilities, and samples. Kathy Pelish, an IIS Editor, interviewed him for this article.

Kathy Pelish: How did the Resource Kit come about, since there wasn't one for prior versions of IIS?

Jim Laurel: We wanted to get the word out about IIS from Microsoft's point of view. There is a growing body of excellent IIS-focused books, but we wanted to add the IIS development team's viewpoint and expertise. IIS is used by a diverse audience: Internet Service Providers (ISPs), content providers and managers, administrators, Web application developers, and IS professionals.

KP: Is the Resource Kit meant as an overview of the product and its technologies, or is it more scenario-specific?

JL: We've found that IIS is used in a range of situations, so the book is organized around specific audiences and scenarios rather than technologies. For example, we devote an entire chapter to the issue of security.

KP: Describe some of the Resource Kit's most useful software utilities...

JL: Well, four utilities come immediately to mind:

  • Web Capacity Analysis Tool (WCAT) runs simulated workloads on client/server configurations. Using WCAT, you can test how IIS and your network configuration respond to a variety of different client requests. The results of these tests can be used to determine the optimal server and network configuration. WCAT is designed to evaluate how Internet servers running Windows NT® Server and IIS respond to various workload simulations. (Ed. note: We're including WCAT with this article as a special free download.)
  • HTTP Monitor was originally developed by the MS.COM team (which knows a thing or two about operating a large Web farm!). It displays "health" statistics and performance metrics to help you determine which servers are running properly.
  • Migration Wizard helps users migrate their current Web-server configuration to IIS 4.0. It consists of a Perl module, the Scavenger, that runs on the source Web-server and creates data files of configuration and user information, Active Server Pages, and COM objects. The Migrator then takes the Scavenger's data files to configure IIS on the target machine.
  • Sign Up Server is a collection of ASP files that allow an ISP to offer unattended sign-up for services on their site.

KP: When can the Web development community expect the book, and where can they obtain it?

JL: The book will be out the first quarter of 1998. The new Microsoft Press site has an entire section dedicated to Windows NT Resource Kits. Barnes and Noble will also carry it.

KP: Will there be a Resource Kit for the next version of Internet Information Server?

JL: Definitely, yes. We want to continue helping our customers get the best out of IIS.

KP: Where can IIS developers send their feedback on the Resource Kit?

JL: We'd like to hear from them at RKINPUT@microsoft.com.

IIS Resource Kit Table of Contents Preview

1. Internet Information Server Overview

The overview chapter describes what's new in IIS 4.0 and outlines the new features and products bundled with the option pack. It also includes a breakdown of the IIS 4.0 architecture and describes how IIS fits into the overall Microsoft Internet strategy.

2. Managing Content

This chapter focuses on the issue of content management. The discussion centers on ways to successfully manage the creation, staging, and deployment of content on intranets and the Internet.

Topics include an overview of content management and the content life cycle. A case study of Microsoft Investor illustrates content-management practices in action.

3. Web Server Capacity Planning

Capacity planning for a Web site involves trying to determine the future needs of the site and providing resources to meet those needs as they arise. Capacity planning is partly an exercise in flexibility, in configuring the site so that it can be expanded in whatever ways become necessary. (This chapter will not address the extensive mathematical modeling techniques that can be used to plan capacity.)

The following topics are covered: hardware and network options, determining your site's requirements, monitoring, scalability, and fault tolerance.

4. Performance Tuning and Optimization

There are many ways to tune and optimize your server, and there are some ways to check (and occasionally even to optimize) the performance of network connections elsewhere that your server uses to provide data to clients. Chapter 4 discusses how to determine your site's needs by estimating, tracking, and testing its performance, and mentions some of the tools available to improve it. It also provides some guidelines to help you optimize your site's performance.

Topics covered in Chapter 4 include: process, memory, and network bottlenecks.

5. Developing Web Applications

This chapter provides an outline of the technological advances that brought about a dramatic change in programming paradigms, how these advances are being applied to Web application development, and how the Web will shape the way you will build an application in the future.

The topics covered include: building on client/server, the role of the client, the role of the server, security ramifications of Web applications, and debugging your application.

6. Data Access and Transactions

Builiding on the multi-tier application development model presented in Chapter 5, Chapter 6 describes how to use Active Server Pages to create dynamic content. This chapter builds on those concepts, introduces the key components of Web driven data access, and discusses how to harness the power of a data driven approach for Web content publishing. Specific topics include: data access using ASP, connecting to a database, and transactional Web applications.

7. IIS and ISPs

Chapter 7 provides a guide for ISPs operating on the NT/IIS platform. Topics covered in Chapter 7 include: remote and automated administration, resource allocation and control, and security.

8. Accessing Legacy Data and Applications

This chapter discusses the development tools and production software packages available to access legacy data and use it in IIS-based Web applications.

9. Migrating to IIS

Chapter 9 is divided into three sections, beginning with a discussion of how to migrate CGI applications to IIS. Section two describes how to migrate a site to IIS from Netscape Enterprise Server, and section three examines the issues involved in migrating from Lotus Domino to IIS.

10. Corporate Standards Planning

Once an enterprise establishes a new business strategy, such as broadening access to business information via an intranet or the Internet, more than powerful software tools will be required to succeed. The organization must establish new standards to ensure that enterprise-wide information sharing is consistent and cost-effective, and that applications are robust, efficient, and scalable.

This chapter discusses the following topics: defining goals for using Web-based technology (focusing on developing standards that support achieving those goals, planning software product standards using a scalable three-tier application architecture, and creating application development standards which promote efficiency, reliability, and consistency.

11. IIS Security

This chapter focuses on the policy, physical, operating system, and IIS-specific levels of security on an IIS site.
Topics include: setting up corporate security policy, how to physically lock a server, Windows NT security issues, and securing active content such as ASP, ISAPI applications, and CGI applications.
